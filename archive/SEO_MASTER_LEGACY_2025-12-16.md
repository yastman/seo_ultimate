## LSI Keywords System (v9.2)

### Принцип

Перед написанием текста **обязательно** читаем `{slug}_clean.json` и создаём LSI-чеклист. Ключи вписываем органично, не спамим.

### Workflow

```
1. Читаем categories/{slug}/data/{slug}_clean.json
2. Извлекаем keywords (primary, secondary, supporting)
3. Извлекаем entity_dictionary (E-E-A-T термины)
4. Создаём LSI-чеклист
5. Пишем текст, вписывая ключи естественно
6. Проверяем coverage (цель: 80%+)
```

### Структура JSON

```json
{
  "keywords": {
    "primary": [...],      // 2-4 вхождения каждого
    "secondary": [...],    // 1-2 вхождения
    "supporting": [...],   // 1 вхождение
    "commercial": [...]    // ТОЛЬКО для meta/CTA
  },
  "entity_dictionary": {
    "product_types": [...],
    "application": [...],
    "benefits": [...],
    "safety": [...]
  }
}
```

### Правила вписывания

| Тип ключа | Где использовать | Частота | Пример |
|-----------|------------------|---------|--------|
| **primary** | H1, intro, H2, текст | 2-4 раза | "чернитель резины" |
| **secondary** | H2, текст, FAQ | 1-2 раза | "полироль для шин" |
| **supporting** | текст, списки | 1 раз | "средство для ухода за резиной" |
| **commercial** | ТОЛЬКО intro CTA + meta | 1 раз | "купить чернитель" |
| **entity_dictionary** | везде как термины | по смыслу | "UV-защита", "pH-нейтральный" |

### LSI-чеклист (шаблон)

Перед написанием создаю список:

```markdown
## LSI Keywords для {slug}

### Primary (2-4 вхождения):
- [ ] ключ 1
- [ ] ключ 2

### Secondary (1-2 вхождения):
- [ ] ключ 3
- [ ] ключ 4

### Supporting (1 вхождение):
- [ ] ключ 5
- [ ] ключ 6

### E-E-A-T термины (по смыслу):
- product_types: ...
- application: ...
- safety: ...
```

### Термины и жаргон (E-E-A-T)

При использовании специализированных терминов — **пояснять при первом упоминании**:

| Термин | Пояснение |
|--------|-----------|
| гритгард | решётка на дне ведра (грязеуловитель) |
| clay bar | глина для очистки |
| iron remover | индикаторный очиститель |
| pre-wash | предварительная мойка |
| гидрофоб | водоотталкивающий эффект |

**Формат:** `термин (пояснение)` при первом использовании, далее — короткий вариант.

### Synonym Rotation (снижение тошноты)

Не повторять одно слово более 3 раз подряд в блоке. Ротация:

| Слово | Синонимы |
|-------|----------|
| мойка | очистка, процедура, обработка |
| шампунь | состав, средство, автошампунь |
| губка | варежка, рукавица, инструмент |
| пена | состав, средство, раствор |

### Чего НЕ делать

- ❌ Не создавать отдельный "LSI-блок" в тексте
- ❌ Не перечислять синонимы списком
- ❌ Не форсировать 100% coverage ценой читабельности
- ❌ Не использовать commercial ключи в инфо-блоках
- ❌ Не повторять один ключ более 4 раз
- ❌ Не использовать жаргон без пояснения

### Пример: cherniteli-shin

**Из JSON:**

- primary: "чернитель резины" (4x), "чернение резины" (3x), "полироль для колес" (2x)
- secondary: "полироль для шин", "средства для чернения шин"
- supporting: "чернитель колёс", "средство для ухода за резиной"
- entity: "глянцевый", "матовый", "защита от UV", "не наносить на протектор"

**В тексте органично:**
> **Чернитель резины** возвращает боковине насыщенный чёрный цвет... Средства для чернения шин бывают глянцевые и матовые... Полироль для колес с защитой от UV продлевает срок службы...

### Coverage Target

| Категория | Keywords | Target Coverage |
|-----------|----------|-----------------|
| Все | primary | 100% |
| Все | secondary | 80%+ |
| Все | supporting | 70%+ |
| Все | commercial | только в CTA |

**Итого: 80%+ overall coverage** — это достаточно. 100% не требуется.

#### Формулы Meta Tags (2025) — E-Commerce Category

**Контекст:** Страницы категорий интернет-магазина. Конкурируем не ценой (это через разметку), а релевантностью + УТП.

**Принцип:** Сначала релевантность (что решает категория), потом УТП магазина. Человек кликает потому что **это точно про его боль**, а УТП добивает решение.

#### Title Tag (50-70 chars)

**Формула:** `{Primary keyword} купить в интернет-магазине | Ultimate`

| Параметр | Значение |
|----------|----------|
| **Длина** | 50-70 символов |
| **Keyword** | Primary keyword в начале |
| **Коммерческий фактор** | "купить в интернет-магазине" или "Купить в" |
| **Бренд** | `| Ultimate` в конце |
| **НЕ добавлять** | Характеристики товаров (UV-защита, глянец, pH и т.д.) |

**Примеры:**

```

❌ Антибитум купить в Киеве | Ultimate (город не evergreen)
❌ Активная пена от 89 грн | Ultimate (цена устареет)
```

#### H1 (заголовок страницы)

**Формула:** `{Primary keyword}` или `{Primary + расширение}`

- НЕ дублировать Title
- БЕЗ "купить"
- Можно добавить уточнение категории

```
Title: Антибитум для авто — удаление битума и смолы | Ultimate
H1: Антибитум и средства для удаления смолы
```

---

#### ❌ Чего НЕ добавлять в Meta Tags

- **Характеристики товаров:** "силиконовый", "глянцевый", "pH-нейтральный" — если неизвестно есть ли в каталоге
- **Конкретные загрязнения:** "удаляет битум, смолу, насекомых" — если не все товары категории это делают
- **Цены:** "от 89 грн" — устаревают
- **Города:** "в Киеве", "цена Украина" — не evergreen
- **Бренды:** конкретные названия производителей
- **Обещания:** "быстрая доставка", "лучшая цена" — сложно доказуемо
- **Keyword stuffing:** перечисление ключевых слов через запятую

#### ✅ Безопасная "польза" (категорийная)

Формулировки, которые верны для **любого** товара категории:

| Категория | Безопасная польза |
|-----------|-------------------|
| Антибитум | "помогает растворить битумные пятна" |
| Чернитель резины | "возвращает цвет боковине" |
| Активная пена | "удаляет грязь без губки" |
| Глина/автоскрабы | "убирает въевшиеся загрязнения после мойки" |
| Очиститель стёкол | "очищает стекло без разводов" |
| Очиститель дисков | "растворяет тормозную пыль" |
| Шампунь для ручной мойки | "создаёт смазку между губкой и лаком" |
| Антимошка | "удаляет следы насекомых" |
| Очиститель шин | "подготавливает резину перед чернением" |

**Слова-помощники:** "помогает", "для", "при правильном применении" — делают фразу честной.

#### ✅ УТП магазина (безопасные)

| Фактор | Формулировка |
|--------|-------------|
| Покупка | "Купить в интернет-магазине Ultimate" |
| Доставка | "Доставка по Украине" |

**НЕ использовать:** "быстрая доставка", "1-2 дня", "лучшая цена", "большой выбор" (если не гарантировано).

#### Правило: Title ≠ H1

```
❌ ПЛОХО:
Title: Антибитум для авто | Ultimate
H1: Антибитум для авто

✅ ХОРОШО:
Title: Антибитум для авто — удаление битума и смолы | Ultimate
H1: Антибитум и средства для удаления смолы
```

#### Почему Google может переписать сниппет

Если на странице:

- Мало уникального текста
- Текст слабее, чем на карточках/фильтрах
- Слишком "шаблонный" description

**Решение:** Текст категории должен содержать те же сущности: "битумные пятна", "смола", "кузов" и т.д.

---

#### Конкуренция с маркетплейсами (Rozetka/Prom)

Маркетплейсы показывают **цены и рейтинг** — это не из description, а из структурированных данных.

**Что нужно:**

- На карточках товаров: цена, наличие, рейтинг/отзывы
- Структурированные данные: `BreadcrumbList`, `ItemList`, `offers`, `aggregateRating`
- Тогда Google покажет "коммерческий" сниппет без цифр в meta

| **Длина**    | 140-155 символов (оптимум для mobile) |
| **CTA**      | В первых 100 символах                 |
| **Keywords** | 2-4 ключа, естественно вписаны        |

**Правила 2025:**

- НЕ ranking factor, но влияет на CTR (+5.8%)
- Google подсвечивает keywords в выдаче
- Фокус на intent, не на плотность ключей
- НЕТ цен в описании (evergreen контент)
- НЕТ "от X грн" — устаревает
- CTA: "Выбирайте", "Закажите", "В каталоге"

**Формула:** `{Что делает продукт}. {Виды/типы}. {CTA + доставка}.`

**Шаблон:**

```
{Primary keyword} {решает проблему}. {Типы}: {перечисление 2-3}.
{CTA} в каталоге Ultimate с доставкой по Украине.
```

**Примеры:**

```
✅ Активная пена смывает грязь без губки за 2 минуты.
   Составы pH 7-13 для любых загрязнений.
   Выбирайте в каталоге Ultimate с доставкой по Украине.
   (155 chars)

✅ Чернитель резины возвращает насыщенный цвет шинам.
   Глянцевые, матовые, с UV-защитой.
   Закажите с доставкой 1-2 дня.
   (142 chars)

❌ Купить активную пену от 89 грн. Доставка по Украине.
   (цена устаревает!)
```

- [ ] Title ≠ H1 (отличается!)
- [ ] Title 50-60 chars
- [ ] Primary keyword в начале Title
- [ ] Description 140-155 chars
- [ ] НЕТ цен ("от X грн")

## КОНСТИТУЦИЯ E-COMMERCE (Evergreen Rules)

Эти правила ПЕРВИЧНЫ. Они определяют долгосрочную ценность текста.

### 1. EVERGREEN КОНТЕНТ

**Текст должен быть актуален через 2 года без правок.**

| Запрещено в тексте   | Разрешено                | Почему                 |
| -------------------- | ------------------------ | ---------------------- |
| "от 150 грн", "500₴" | "цена зависит от объёма" | Цены меняются          |
| "доставка 1-2 дня"   | "быстрая доставка"       | Сроки не гарантированы |
| "акция до 31.12"     | —                        | Акции заканчиваются    |
| "в 2024 году"        | —                        | Устаревает             |

**Где можно цифры:** Только Meta Description (для CTR) и технические характеристики (pH 7-13, разведение 1:50).

### 2. CTA ДОЗИРОВАНО

| Секция           | CTA разрешён? | Пример                                         |
| ---------------- | ------------- | ---------------------------------------------- |
| INTRO            | ✅ ДА         | "В каталоге Ultimate — с доставкой по Украине" |
| Как выбрать      | ❌ Инфо       | Критерии без продаж                            |
| Виды/Ассортимент | ❌ Инфо       | Типы с внутренними ссылками                    |
| Таблица          | ❌ Инфо       | Характеристики без CTA                         |
| FAQ (Q1-Q4)      | ❌ Инфо       | Ответы по существу                             |
| FAQ (последний)  | ✅ ДА         | "Выбирайте в каталоге Ultimate"                |

### 4. E-E-A-T ЧЕРЕЗ ЭКСПЕРТИЗУ

**Обязательно 3-7 терминов из entity_dictionary:**

- pH уровни (нейтральный 7-9, щелочной 11-13)
- Разведение (1:30, 1:50, 1:100)
- Оборудование (пенокомплект, АВД, давление)
- Время экспозиции (2-3 минуты)
- Безопасность (не допускать высыхания)

**Плюс 1-2 экспертных совета** (не вода!):

```markdown
✅ "Начинайте с разведения 1:50 — для сильных загрязнений уменьшите до 1:30"
✅ "Не допускайте высыхания пены на кузове — смывайте сверху вниз"
```

### 4.1 E-E-A-T ACCURACY (Точность фактов)

**Универсальные правила для ВСЕХ категорий автохимии.**

**1. Консистентность данных:**
Технические характеристики ОДИНАКОВЫЕ по всему тексту:

- Intro = Список = Таблица = FAQ
- Если в intro "pH 7-9", то везде "pH 7-9" (не "6-8" в таблице)

**2. Оговорки для переменных значений:**

| Тип данных | Формулировка |
|------------|--------------|
| Концентрация/разведение | "X:Y — уточняйте в инструкции производителя" |
| Время действия | "обычно X минут" / "от X до Y минут" |
| Температура | "рекомендуется X°C" |
| Расход | "примерно X мл/м²" |

**3. Смягчение категоричности:**

| Плохо | Хорошо |
|-------|--------|
| "удаляет всё" | "эффективен против..." |
| "минимум X" | "от X" / "рекомендуется X" |
| "за одно применение" | "при правильном применении" |
| "лучший на рынке" | убрать |
| "гарантированно" | "как правило" |

**4. Специализация продуктов:**

```
Каждый продукт решает СВОЮ задачу.
НЕ обещать универсальность!

Если загрязнение требует спец. средства → рекомендуй его:
"Для [сложное загрязнение] используйте специальный [категория]"
```

**5. Self-check (универсальный):**

- [ ] Тех. данные одинаковые везде?
- [ ] Переменные значения с оговорками?
- [ ] Нет категоричных обещаний?
- [ ] Продукт не "решает всё" (отдельно про битум/диски)?

### 4.2 ENVIRONMENTAL FACTORS (Факторы среды)

**Обязательно учитывать украинские реалии:**

1. **Жесткая вода:**

    - "В условиях жесткой воды увеличьте концентрацию на 10-15%..."
    - "Используйте составы с комплексообразователями..."

2. **Сезонность:**

    - "Зимой (реагенты) используйте щелочные составы..."
    - "Летом (насекомые) требуется предварительное замачивание..."

3. **Безопасность (Safety First):**
    - "Не наносить на горячий кузов..."
    - "Смывать сверху вниз..."

### 4.3 РЕДАКТИРОВАНИЕ ТЕКСТА (Edit vs Write)

**ПРАВИЛО: Редактируй кусками, НЕ переписывай весь файл!**

| Действие    | Инструмент | Когда                         |
| ----------- | ---------- | ----------------------------- |
| Новый файл  | `Write`    | Только при создании с нуля    |
| Исправление | `Edit`     | Точечные правки существующего |
| Доработка   | `Edit`     | Улучшение после валидации     |

**Почему Edit, а не Write:**

- Контроль: видно что именно изменилось
- Безопасность: не потеряешь удачные части
- История: легче откатить если что-то сломалось
- Скорость: меньше токенов на операцию

**Workflow исправлений:**

```
1. Валидация → выявлены проблемы
2. Читаешь файл (Read)
3. Точечные Edit для каждой проблемы
4. Повторная валидация
5. Если ОК → готово
6. Если нет → ещё Edit (не Write!)
```

**Исключение:** Полная переработка структуры (редко) — тогда Write допустим.

### 5. КЛЮЧИ = КЛАСТЕРЫ

**10-15 осмысленных ключей достаточно** (из \_clean.json).

| Подход               | Результат                      |
| -------------------- | ------------------------------ |
| ❌ 52 ключа = спам   | Density >5%, читать невозможно |
| ❌ 12/12 любой ценой | Натужные вхождения             |
| ✅ 10-12 естественно | Coverage 80%+, текст читается  |

**Правило:** Лучше 10/12 с density 1.5%, чем 12/12 с density 4%.

## СТРОГИЕ ЗАПРЕТЫ (3 правила)

### 1. PRICE BAN (Анти-Цена)

**Запрещено:** Указывать конкретные цифры в гривнах/долларах в теле текста и FAQ.

| Контекст                 | Статус       | Пример                                 |
| ------------------------ | ------------ | -------------------------------------- |
| Тело текста              | ❌ ЗАПРЕЩЕНО | "Цена от 150 грн", "Стоимость 500 грн" |
| FAQ ответы               | ❌ ЗАПРЕЩЕНО | "...стоит около 200 грн"               |
| Таблицы (колонка "Цена") | ❌ ЗАПРЕЩЕНО | "от 150 грн"                           |
| **Meta Description**     | ❌ ЗАПРЕЩЕНО | ~~"от 89 грн"~~ — цены устаревают, evergreen контент |

**Почему:** Цены меняются → текст устаревает → потеря доверия Google (E-E-A-T).

**Разрешено:** Слово "цена" без цифр: "выгодная цена", "доступная цена", "цена в каталоге".

### 2. STOP-PHRASES (Анти-Флафф расширенный)

**Запрещено:** Использовать вводные конструкции и "воду":

```
❌ "В современном мире..."
❌ "Каждый автомобилист знает..."
❌ "На сегодняшний день..."
❌ "Широкий ассортимент..."
❌ "Индивидуальный подход..."
❌ "Лучшее качество..."
❌ "Компания Ultimate рада предложить вам..."
```

**Решение:** Начинать сразу с сути — проблема или решение.

### 3. FAKE CTA (Анти-Спам)

**Запрещено:** Вставлять призывы "купите у нас" в середине информационных блоков.

| Место                 | CTA разрешён? |
| --------------------- | ------------- |
| INTRO (первый абзац)  | ✅ ДА         |
| H2: Как выбрать       | ❌ НЕТ        |
| H2: Виды/Ассортимент  | ❌ НЕТ        |
| H2: Таблица           | ❌ НЕТ        |
| FAQ (ответы Q1-Q4)    | ❌ НЕТ        |
| FAQ (последний ответ) | ✅ ДА         |

**Примеры запрещённого:**

```
❌ В FAQ: "...поэтому купите наш шампунь прямо сейчас!"
❌ В характеристиках: "...pH 12, который можно заказать онлайн"
❌ В "Как выбрать": "...и закажите в Ultimate с доставкой"
```
