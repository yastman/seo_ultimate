# UK Content Generator v1.0

## Quick Start

```
/uk-content-generator {slug}
```

**Input:** `uk/categories/{slug}/data/{slug}_clean.json` + `uk/categories/{slug}/meta/{slug}_meta.json` + research/*.md
**Output:** `uk/categories/{slug}/content/{slug}_uk.md`

---

## ⚠️ ПЕРШИЙ КРОК: Визначити тип сторінки

Перед написанням контенту **ОБОВ'ЯЗКОВО** перевірити `parent_id` в `_clean.json`:

```json
// Hub Page (L1) — батьківська категорія
{ "parent_id": null }

// Product Page (L2/L3) — товарна категорія
{ "parent_id": "aksessuary" }
```

| Тип | Умова | Шаблон |
|-----|-------|--------|
| **Hub Page** | `parent_id = null` | Див. [../content-generator/references/hub-pages.md](../content-generator/references/hub-pages.md) |
| **Product Page** | `parent_id ≠ null` | Buyer Guide (нижче) |

## ⚠️ Комерційний vs Інфо інтент

Buyer guide = допомога у **виборі та покупці**, НЕ інструкція із застосування.

### Що допустимо (комерційний інтент)

| Елемент | Приклад |
|---------|---------|
| Таблиця порівняння типів | "Нейтральний vs кислотний — для кого" |
| Сценарії вибору | "Якщо авто з керамікою → нейтральний" |
| Критерії на етикетці | "pH-neutral = безпечний для захисту" |
| Згадування профтермінів | "двофазне миття", "метод двох відер" |
| FAQ про вибір | "Який об'єм обрати?", "Чим відрізняється від X?" |

### Що НЕ допустимо (інфо-інтент → канібалізація блогу)

| Елемент | Чому не можна |
|---------|---------------|
| Покрокові інструкції (5+ кроків) | How-to = інфо-інтент |
| "Як мити машину" | Це стаття для блогу |
| Детальні помилки застосування | Занадто глибоко в процес |

---

## СТОП-ЛИСТ: Заборонені назви секцій

**BLOCKER:** Ці H2/H3 = how-to контент. НЕ використовувати!

| ❌ Заборонено | Чому | ✅ Альтернатива |
|--------------|------|-----------------|
| "Як наносити {X}" | How-to інструкція | Видалити або "Що врахувати при виборі" |
| "Професійний підхід до нанесення" | Інструкція, не вибір | "Що впливає на результат" |
| "Техніка застосування" | How-to | Видалити секцію |
| "Підготовка поверхні" | How-to | 1 фраза в intro максимум |
| "Помилки при нанесенні" | Інфо-інтент | "На що дивитися на етикетці" |
| "Покрокова інструкція" | How-to | Видалити повністю |

## Режим "Без пруфів" (за замовчуванням)

### Що НЕ вставляти в текст

| Заборонено | Чому |
|------------|------|
| Посилання, цитати, [1][2] | Buyer guide, не наукова стаття |
| Відсотки ("56% детейлерів...") | Неперевірювано |
| Точні дослідження | "Згідно з дослідженням X..." — ні |
| Бренди в прикладах | Нейтральність |

### Спірні теми → пом'якшити або видалити

| Тема | ❌ Було | ✅ Стало |
|------|--------|---------|
| 6PPD/антиозонанти | "Чорнило вимиває 6PPD" | Видалити повністю |
| "Сушить гуму" | "Силікон сушить гуму" | "Деякі склади можуть з часом впливати на еластичність" |
| TPMS | "Пошкодить датчики" | "При нанесенні уникати зони датчиків" |

## Workflow

```
1. Read data        → uk/categories/{slug}/data/{slug}_clean.json (keywords, entities, synonyms)
2. Check parent_id  → null = Hub Page, інакше = Product Page
3. If Hub Page      → читати references/hub-pages.md
4. Read meta        → uk/categories/{slug}/meta/{slug}_meta.json (H1 + keywords_in_content)
5. Find research    → uk/categories/{slug}/research/RESEARCH_DATA.md (як довідка + FAQ)
6. Write content    → Hub: за процесами | Product: buyer guide (комерційний інтент!)
7. Check H2         → мінімум 1 H2 містить secondary keyword
8. Validate         → validate_uk.py
```

---

## LSI/Семантика з meta.json

### Джерело

`uk/categories/{slug}/meta/{slug}_meta.json` → `keywords_in_content`:
- `primary` — основний ключ (в H1, intro, 2-3x в тексті)
- `secondary` — 1x природне входження кожного
- `supporting` — 1-2 ключових, без "комерційних" (купити, ціна, замовити)

### Куди вставляти LSI

| Місце | Пріоритет | Приклад |
|-------|-----------|---------|
| Intro | Високий | Primary + 1 secondary |
| Підзаголовки H2/H3 | Високий | Secondary в заголовках секцій |
| Таблиці | Середній | Supporting в комірках |
| FAQ питання | Середній | Secondary/supporting у формулюваннях |
| Підсумок | Низький | Primary |

## Розподіл ключів по заголовках

| Заголовок | Що має бути | Обов'язково | Приклад |
|-----------|-------------|-------------|---------|
| H1 | Primary keyword | ✅ BLOCKER | "Автошампунь для ручного миття" |
| H2 (мінімум 1) | **Secondary keyword** | ✅ WARNING | "Як обрати шампунь для ручного миття авто" |
| H3 | Supporting | INFO | "Місце у двофазному митті" |

**⚠️ H1 БЕЗ "Купити"** — "Купити" тільки в Title!

## Структура контенту (Buyer Guide)

```markdown
# {H1 з meta — БЕЗ "Купити"}

{Intro: що це + навіщо + ключове правило + 1 фраза про вибір фінішу}

## Як обрати {категорію}

| Тип | Особливості | Для кого |
|-----|-------------|----------|

**Сценарії:**
- **Якщо {X}** → обирай {Y}

### На що дивитися на етикетці *(опціонально)*

| Маркер | Що означає | Кому підходить |
|--------|------------|----------------|

## Особливості використання *(коротко, 1-2 абзаци)*

{Професійний термін/метод} — {що це і чому важливо для вибору}.

**НЕ писати:** покрокові інструкції (how-to = інфо-інтент).

## Що впливає на стійкість *(для дресингів/покриттів)*

| Фактор | Вплив |
|--------|-------|

## FAQ

### {Питання про вибір}?
{Відповідь 2-3 речення}

### Обов'язкові FAQ за типами категорій

**Для догляд/дресинги/захист:**
- "Скільки тримається?" → відповідь через умови
- "Як уникнути розбризкування (sling)?" → тонкий шар, аплікатор, пауза

**Для шин:**
- Завжди згадати: "тільки бічна частина, не протектор"
- "Чому шина коричневіє?" → окислення, старий шар, рішення

---

**Підсумок:**
- Якщо {сценарій 1}: {рекомендація}
```

## Validation Checklist

### Структура
- [ ] H1 без "Купити"
- [ ] Intro 30-60 слів
- [ ] Таблиця порівняння типів
- [ ] FAQ 3-5 питань
- [ ] **НЕМАЄ how-to інструкцій**

### SEO/LSI
- [ ] Primary keyword в H1 та intro
- [ ] Мінімум 1 H2 містить primary або secondary keyword
- [ ] Secondary keywords — 1x кожен, природно
- [ ] Без "комерційних" ключів у body (купити, ціна, замовити)

### UK-специфічне
- [ ] **Термінологія UK** (гума не резина, миття не мойка)
- [ ] **Title містить "Купити"**
- [ ] **H1 БЕЗ "Купити"**
- [ ] Природна українська мова (не калька з російської)

### Якість контенту
- [ ] **Немає спірних тверджень без пом'якшення**
- [ ] **Немає брендів/відсотків/точних цифр**
- [ ] **UK-first:** український термін першим, англ. у дужках
