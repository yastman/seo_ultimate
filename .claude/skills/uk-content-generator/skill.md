# UK Content Generator v3.3

## Quick Start

```
/uk-content-generator {slug}
```

**Input:** `uk/categories/{slug}/data/{slug}_clean.json` + `uk/categories/{slug}/meta/{slug}_meta.json` + research/*.md (як довідка)
**Output:** `uk/categories/{slug}/content/{slug}_uk.md`

---

## ПЕРШИЙ КРОК: Визначити тип сторінки

Перед написанням контенту **ОБОВ'ЯЗКОВО** перевірити `parent_id` в `_clean.json`:

```json
// Hub Page (L1) — батьківська категорія
{ "parent_id": null }

// Product Page (L2/L3) — товарна категорія
{ "parent_id": "aksessuary" }
```

| Тип | Умова | Шаблон |
|-----|-------|--------|
| **Hub Page** | `parent_id = null` | Див. [references/hub-pages.md](references/hub-pages.md) |
| **Product Page** | `parent_id ≠ null` | Buyer Guide (нижче) |

**Для Hub Page:**
- H2 за процесами + secondary keyword, НЕ за типами товарів
- Без детальних гайдів (канібалізація L2!)
- FAQ з RESEARCH_DATA.md або шаблони (питання про екосистему)
- 2000-2500 знаків б/п

## Комерційний vs Інфо інтент

Buyer guide = допомога у **виборі та покупці**, НЕ інструкція із застосування.

### Що допустимо (комерційний інтент)

| Елемент | Приклад |
|---------|---------|
| Таблиця порівняння типів | "Нейтральний vs кислотний — для кого" |
| Сценарії вибору | "Якщо авто з керамікою → нейтральний" |
| Критерії на етикетці | "pH-neutral = безпечний для захисту" |
| Згадування профтермінів | "двофазне миття", "метод двох відер" |
| FAQ про вибір | "Який об'єм обрати?", "Чим відрізняється від X?" |

### Що НЕ допустимо (інфо-інтент → канібалізація блогу)

| Елемент | Чому не можна |
|---------|---------------|
| Покрокові інструкції (5+ кроків) | How-to = інфо-інтент |
| "Як мити машину" | Це стаття для блогу |
| Детальні помилки застосування | Занадто глибоко в процес |
| "Візьміть відро, налийте воду..." | Інструкція, не допомога у виборі |

### Як згадувати застосування (правильно)

```markdown
❌ НЕПРАВИЛЬНО (how-to):
## Як застосовувати
1. Підготуйте два відра
2. Змочіть рукавицю
3. Вимийте панель
4. Сполосніть
5. Повторіть

✅ ПРАВИЛЬНО (експертне згадування):
**Метод двох відер** — професійний підхід: одне відро
з шампунем, друге для полоскання рукавиці. Бруд не
повертається в розчин — менше ризику подряпати лак.
```

**Правило:** Професійні терміни згадувати як контекст для вибору, не як інструкцію.

## СТОП-ЛИСТ: Заборонені назви секцій

**BLOCKER:** Ці H2/H3 = how-to контент. НЕ використовувати!

| ❌ Заборонено | Чому | ✅ Альтернатива |
|--------------|------|-----------------|
| "Як наносити {X}" | How-to інструкція | Видалити або "Що врахувати при виборі" |
| "Професійний підхід до нанесення" | Інструкція, не вибір | "Що впливає на результат" |
| "Техніка застосування" | How-to | Видалити секцію |
| "Підготовка поверхні" | How-to | 1 фраза в intro максимум |
| "Помилки при нанесенні" | Інфо-інтент | "На що дивитися на етикетці" |
| "Покрокова інструкція" | How-to | Видалити повністю |
| "Правильне використання" | How-to | Видалити секцію |

**Правило:** Секція описує ПРОЦЕС (як робити) → how-to → ВИДАЛИТИ.
Buyer guide описує ВИБІР (що взяти) → комерційний інтент → ЗАЛИШИТИ.

## Режим "Без пруфів" (за замовчуванням)

### Що НЕ вставляти в текст

| Заборонено | Чому |
|------------|------|
| Посилання, цитати, [1][2] | Buyer guide, не наукова стаття |
| Відсотки ("56% детейлерів...") | Неперевірювано |
| Точні дослідження | "Згідно з дослідженням X..." — ні |
| Бренди в прикладах | Нейтральність |

### Спірні теми → пом'якшити або видалити

| Тема | ❌ Було | ✅ Стало |
|------|--------|---------|
| 6PPD/антиозонанти | "Чорнило вимиває 6PPD" | Видалити повністю |
| "Сушить гуму" | "Силікон сушить гуму" | "Деякі склади можуть з часом впливати на еластичність" |
| TPMS | "Пошкодить датчики" | "При нанесенні уникати зони датчиків" |
| Гальмівний шлях | "Збільшує на X%" | "Уникати потрапляння на протектор" |
| Шкода/безпека | Категоричні заяви | "може", "ризик вищий при...", "залежить від..." |

## Workflow

```
1. Read data        → uk/categories/{slug}/data/{slug}_clean.json (keywords, synonyms)
2. Check parent_id  → null = Hub Page, інакше = Product Page
3. If Hub Page      → читати references/hub-pages.md, знайти підкатегорії в catalog_structure.json
4. Read meta        → uk/categories/{slug}/meta/{slug}_meta.json (H1 + keywords_in_content: primary/secondary/supporting)
5. Find research    → uk/categories/{slug}/research/RESEARCH_DATA.md (як довідка + FAQ, не копіювати!)
6. Write content    → Hub: за процесами | Product: buyer guide (комерційний інтент!)
7. Check H2         → мінімум 2 H2 містять secondary keyword
8. Check density    → check_keyword_density.py --lang uk (stem ≤2.5%)
9. Check nausea     → check_water_natasha.py (тошнота ≤3.5)
10. Check SEO       → check_seo_structure.py (H2 keywords, intro check)
11. Fix spam        → замінити тематичні слова на синоніми
12. Validate        → validate_content.py --mode seo
```

**Для Hub Page (крок 3):**
```bash
# Знайти підкатегорії
cat data/catalog_structure.json | python3 -c "
import json, sys
data = json.load(sys.stdin)
children = [c for c in data if c.get('parent_id') == '{slug}']
for c in children: print(f\"{c['id']}: {c['name']}\")
"
```

**Пошук research файлу:**
1. Якщо є `RESEARCH_DATA.md` з контентом (>5KB) — використовувати його
2. Інакше — знайти `research_data_*.md` як "підказку"
3. **КРИТИЧНО:** Не копіювати спірні факти/цифри з research!

## LSI/Семантика з meta.json

### Джерело

`uk/categories/{slug}/meta/{slug}_meta.json` → `keywords_in_content`:
- `primary` — основний ключ (в H1, intro, 2-3x в тексті)
- `secondary` — 1x природне входження кожного
- `supporting` — 1-2 ключових, без "комерційних" (купити, ціна, замовити)

### Куди вставляти LSI

| Місце | Пріоритет | Приклад |
|-------|-----------|---------|
| Intro | Високий | Primary + 1 secondary |
| Підзаголовки H2/H3 | Високий | Secondary в заголовках секцій |
| Таблиці | Середній | Supporting в комірках |
| FAQ питання | Середній | Secondary/supporting у формулюваннях |
| Підсумок | Низький | Primary |

**Антипатерн:** Не "набивати" всі ключі в одному абзаці — розподіляти природно.

### Розподіл ключів по заголовках

| Заголовок | Що має бути | Обов'язково | Приклад |
|-----------|-------------|-------------|---------|
| H1 | Primary keyword | ✅ BLOCKER | "Автошампунь для ручного миття" |
| H2 (мінімум 2) | **Secondary keyword** | ✅ WARNING | "Як обрати шампунь для ручного миття авто" |
| H3 | Supporting | INFO | "Місце у двофазному митті" |
| FAQ питання | Про вибір (з RESEARCH_DATA.md) | INFO | "Який шампунь обрати для авто з керамікою?" |

**Правило:** Мінімум 2 H2 МАЮТЬ містити secondary keyword з `_meta.json`.

## Валідація

### Primary keyword

Брати з одного з джерел:
1. `uk/categories/{slug}/data/{slug}_clean.json` → перший ключ за volume
2. `uk/categories/{slug}/meta/{slug}_meta.json` → поле `h1` (без "Купити")

### Команда валідації

```bash
python3 scripts/validate_content.py uk/categories/{slug}/content/{slug}_uk.md "{primary_keyword}" --mode seo
```

**Приклад для chorniteli-shin:**
```bash
python3 scripts/validate_content.py uk/categories/chorniteli-shin/content/chorniteli-shin_uk.md "чорнитель гуми" --mode seo
```

### Відомі обмеження

| Обмеження | Що робити |
|-----------|-----------|
| Grammar check потребує Java | Не блокер — пропускається автоматично |
| `--mode quality` дає WARNING по "Water" | Для SEO-категорій не критично |
| Фінальний гейт пайплайну | Використовувати `--mode seo` |

## Структура контенту (Buyer Guide)

```markdown
# {H1 з meta — БЕЗ "Купити"}

{Intro: що це + навіщо + ключове правило + 1 фраза про вибір фінішу}

**Приклади intro:**

```
❌ ПОГАНО (енциклопедія):
Віск для автомобіля — захисне покриття, яке створює
бар'єр між лаком та зовнішнім середовищем. Основні завдання:
захист від UV-випромінювання, окислення та бруду.

✅ ДОБРЕ (buyer guide):
Віск захищає кузов від сонця, бруду та дрібних подряпин,
при цьому надає глибокий блиск. Якщо вам важливий теплий
класичний фініш — обирайте карнаубу, для тривалого
захисту без частого оновлення — синтетику.
```

**Ключові відмінності:**
- ❌ Погано: визначення "X — це Y, яке..."
- ✅ Добре: одразу користь + сценарій вибору
- ✅ Добре: звернення до читача ("вам", "обирайте")

| Фактор | Вплив |
|--------|-------|

## FAQ

{Відповідь 2-3 речення}

### Обов'язкові FAQ за типами категорій

**Для догляд/дресинги/захист:**
- "Скільки тримається?" → відповідь через умови (основа, підготовка, погода)
- "Як уникнути розбризкування (sling)?" → тонкий шар, аплікатор, пауза
- "Чи можна використовувати взимку?" → умови нанесення важливіші за тип

**Для шин:**
- Завжди згадати: "тільки бічна частина, не протектор" (без страшилок, але явно)
- "Чому шина коричневіє?" → окислення, старий шар, рішення

**Для захисних покриттів:**
- "Скільки шарів наносити?"
- "Чи можна поверх кераміки/воску?"

**ВАЖЛИВО: FAQ не має дублювати таблиці!**

| ❌ Дублювання | ✅ Унікальне питання |
|--------------|---------------------|
| "Чим відрізняється натуральний від синтетичного?" (вже в таблиці) | "Чи можна наносити поверх кераміки?" |
| "Які бувають форми випуску?" (вже в таблиці) | "Як зрозуміти що пора оновити захист?" |

**Правило:** Кожен FAQ = унікальна інформація, якої НЕМАЄ в таблицях.
```

## Контроль тошноти та переспаму

### Порогові значення

| Метрика | Ціль | Блокер |
|---------|------|--------|
| Stem-група ключа | ≤2.5% | >3.0% = SPAM |
| Класична тошнота | ≤3.5 | >4.0 |
| Вода | 40-65% | >75% |
| **Академічна тошнота** | **≥7%** | **<6% = сухий текст** |

### Живість тексту

Якщо академічна тошнота <7% — текст "сухий", читається як довідник.

**Як виправити:**
- Додати звернення до читача: "вам підійде", "якщо ви новачок"
- Додати зв'язуючі фрази: "при цьому", "важливо враховувати"
- Більше сценаріїв "Якщо X → Y"
- Менше номінативних конструкцій ("захист кузова" → "захищає кузов")

### Команди перевірки

```bash
# Щільність ключів та переспам
python3 scripts/check_keyword_density.py uk/categories/{slug}/content/{slug}_uk.md --lang uk

# Тошнота та вода (Natasha)
python3 scripts/check_water_natasha.py uk/categories/{slug}/content/{slug}_uk.md
```

### Як уникнути переспаму

**Проблема:** Stem-група (машин*, акумулятор*) перевищує 2.5%

**Рішення:** Замінити частину входжень на синоніми.

**Правило:** Цільові ключі з `_clean.json` НЕ заміняти. Заміняти лише тематичні слова, які не є ключами.

### LSI-синоніми для розбавлення

**Інструменти/обладнання:**
| Слово | Синоніми |
|-------|----------|
| машинка/машина | інструмент, пристрій, варіант, модель |
| акумулятор | АКБ, елемент живлення, джерело живлення |
| батарея | АКБ, заряд, Li-Ion елемент |
| мережева | дротова, від розетки |
| бездротова | автономна, мобільна |

**Автохімія/детейлінг:**
| Слово | Синоніми |
|-------|----------|
| засіб | склад, продукт, препарат |
| нанесення | обробка, застосування |
| поверхня | покриття, основа, матеріал |
| захист | бар'єр, шар, покриття |
| блиск | глянець, сяйво, фініш |
| чищення | очищення, видалення, обробка |

**Автомобіль:**
| Слово | Синоніми |
|-------|----------|
| автомобіль | авто, машина, транспорт |
| кузов | поверхня, панелі, деталі |
| колесо | диск, шина (контекстно) |
| салон | інтер'єр, внутрішня частина |

**Процеси:**
| Слово | Синоніми |
|-------|----------|
| робота | процес, використання, експлуатація |
| полірування | обробка, корекція (контекстно) |
| миття | очищення, догляд |

### Як знизити тошноту

Якщо класична тошнота >3.5 (слово X повторюється занадто часто):

1. Визначити винуватця через `check_water_natasha.py`
2. Якщо це НЕ цільовий ключ — замінити частину на синоніми
3. Якщо це цільовий ключ — додати LSI-слова для розбавлення (збільшити загальну к-ть слів)

**Приклад:** "батарея" 13 разів → замінити 4 на "АКБ", "заряд", "Li-Ion елемент" → тошнота 3.61 → 3.32 ✅

## Self-Check перед збереженням

**ОБОВ'ЯЗКОВО** перед Write виконати цю перевірку:

### Головне питання

Перечитай кожну секцію і запитай:
> "Ця секція допомагає ОБРАТИ продукт чи ВЧИТЬ його застосовувати?"

- **Допомагає обрати** → залишити
- **Вчить застосовувати** → ВИДАЛИТИ (це how-to)

### Чек-лист самоперевірки

| Перевірка | Що шукати | Дія якщо знайдено |
|-----------|-----------|-------------------|
| How-to секції | H2/H3 зі СТОП-ЛИСТА | Видалити секцію |
| Сухий текст | Немає "вам", "якщо ви", "вам підійде" | Додати звернення |
| Intro-визначення | "X — це Y, яке..." | Переписати з фокусом на вибір |
| FAQ дублює таблицю | Те саме питання що в H2 | Замінити питання |
| Енциклопедичний тон | Немає сценаріїв "Якщо X → Y" | Додати 3+ сценарії |

### Червоні прапорці в тексті

Якщо бачиш ці фрази — це how-to, видали:
- "Перед нанесенням необхідно..."
- "Рекомендується повне миття..."
- "Дайте висохнути протягом..."
- "Повне затвердіння відбувається..."
- "Використовуйте тільки pH-нейтральні..."
