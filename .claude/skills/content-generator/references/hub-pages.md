# Hub Pages (L1 и L2 с дочерними)

## SEO-стандарты 2025–2026

**Helpful Content System** пессимизирует страницы с "водой" ради ключевых слов.

Hub Page = **диспетчер**, не энциклопедия. Задача: помочь выбрать направление, а не читать лекцию.

| Факт | Следствие |
|------|-----------|
| 66% топовых страниц <400 слов | Длинный текст ≠ качественный |
| Mobile First | 1000 слов = бесконечный скролл |
| Интент = навигация | Человек выбирает направление, не покупает |

---

## Определение

**Hub Page** — категория, объединяющая дочерние подкатегории. Работает как "диспетчер": разводит трафик по схеме **Проблема → Решение (ссылка)**.

## Два типа хабов

| Тип | Определение | Интент | Пример |
|-----|-------------|--------|--------|
| **L1 Hub** | `parent_id = null` | Навигационный | aksessuary, moyka-i-eksterer |
| **L2 Mini-Hub** | `parent_id ≠ null` + есть дочерние L3 | Коммерческий + навигационный | sredstva-dlya-kozhi, avtoshampuni |

## Как определить тип страницы

```
1. Читаем _clean.json → parent_id
   │
   ├─ null → L1 HUB
   │
   └─ not null → проверяем дочерние
                 │
                 ├─ есть дочерние → L2 MINI-HUB
                 │
                 └─ нет дочерних → BUYER GUIDE
```

**Где смотреть дочерние:**
- `tasks/reports/STRUCTURE_TREE.md` — визуальное дерево
- `data/catalog_structure.json` — JSON со всеми связями

## ⛔️ СТОП-ФАКТОРЫ для Hub Pages

### Запрет на каннибализацию

НЕ писать подробные гайды по подкатегориям:
- ❌ "Как выбрать микрофибру" → контент для L2 mikrofibra-i-tryapki
- ❌ "Виды кистей" → контент для L2 shchetki-i-kisti
- ❌ "Как стирать микрофибру" → FAQ для L2, не для Hub

### Запрет на H2 с прямыми ключами подкатегорий

| ❌ Плохо | ✅ Хорошо |
|---------|----------|
| H2: Микрофибра | H2: Текстиль для сушки и полировки |
| H2: Ведра | H2: Организация безопасной мойки |
| H2: Щётки и кисти | H2: Детейлинг колёс и сложных зон |

### Запрет на "воду"

Никакой "воды" ради объёма. Пишем только по схеме **"Проблема → Решение (Ссылка)"**.

## Параметры Hub Page (2025–2026)

| Параметр | L1 Hub | L2 Mini-Hub |
|----------|--------|-------------|
| **Общий объём** | 400-500 слов | 400-500 слов |
| **Lead (Intro)** | 50-70 слов | 50-70 слов |
| **Body** | 250-300 слов | 300-400 слов |
| **FAQ** | 100-130 слов (2-3 вопроса) | 100-150 слов (2-3 вопроса) |
| **H2** | По процессам, НЕ по товарам | По процессам/системе |
| **Плотность** | ≤2.0% | ≤2.5% |

## Структура Hub Page

### Lead (50-70 слов) — НАД плитками категорий

**Задача:** Объяснить, куда попал пользователь, помочь выбрать направление.

```markdown
# {H1 — название категории, БЕЗ "Купить"}

{Что это за группа} + {главная польза} + {3 направления для выбора}.
Выберите проблему ниже, чтобы найти решение.
```

**Пример:**
> Чистое стекло — это безопасность. Здесь собраны средства для трёх этапов: глубокая очистка (полироли), регулярная мойка (очистители) и защита от воды (антидождь). Выберите задачу ниже.

### Body (250-400 слов) — ПОД плитками категорий

**Задача:** "Гид по решению проблем" — спасает от статуса "Thin Content".

**Формат:** Проблема → Решение (упоминание подкатегории)

```markdown
## {H2 по проблеме/процессу, НЕ по типу товара}

{Описание проблемы 1-2 предложения}

**Если {проблема A}** → {решение} (упоминание L3)
**Если {проблема B}** → {решение} (упоминание L3)

| Проблема | Решение | Куда смотреть |
|----------|---------|---------------|
| ... | ... | {название подкатегории} |
```

### FAQ (100-150 слов) — в самом НИЗУ

**Задача:** Захватить PAA (People Also Ask) запросы.

```markdown
## FAQ

### {micro_intent — общий вопрос про экосистему}?
{Ответ 2-3 предложения, направляющий к подкатегории}

### {micro_intent — вопрос про последовательность/выбор направления}?
{Ответ 2-3 предложения}
```

## Пример: sredstva-dlya-stekol (L2 Mini-Hub)

**Дочерние L3:** omyvatel, antidozhd, ochistiteli-stekol

### Lead (65 слов)

> Чистое стекло — это безопасность и комфорт. Три типа задач требуют разных средств: удаление стойких загрязнений (полироли и очистители), регулярная мойка в дороге (омыватели) и защита от воды при скорости (антидождь). Каждый этап решает свою проблему — выберите задачу ниже.

### Body (300 слов)

```markdown
## Когда стекло требует глубокой очистки

Масляная плёнка, следы от дворников, налёт от реагентов — обычный омыватель не справится.

**Если видны разводы после мойки** → абразивный очиститель или полироль для стекла
**Если дворники скрипят** → причина часто в загрязнении, не в самих дворниках

| Проблема | Причина | Решение |
|----------|---------|---------|
| Радужные разводы | Масляная плёнка | Очиститель стёкол |
| Белёсый налёт | Известь от воды | Кислотный состав |
| Царапины от дворников | Микрочастицы | Полироль |

## Защита от воды на скорости

При 60+ км/ч обработанное стекло отталкивает капли — дворники почти не нужны.

**Если часто ездите в дождь** → антидождь на основе силикона (держится 2-3 месяца)
**Если редко обрабатываете** → керамический состав (держится до 6 месяцев)

## Омыватели: не только для зимы

**Если -25°C и ниже** → концентрат с разведением под температуру
**Если лето** → антимошка убирает насекомых без разводов
```

### FAQ (120 слов)

```markdown
## FAQ

### Что сначала — очистка или антидождь?
Сначала очистка. Антидождь наносится только на чистое обезжиренное стекло. Иначе состав ляжет неравномерно и быстро смоется.

### Почему омыватель не убирает разводы?
Омыватель — для регулярной очистки в дороге. Стойкая плёнка (масло, реагенты) требует специального очистителя с абразивом или кислотой. Омыватель просто размажет загрязнение.
```
