# Hub Pages (L1 и L2 с дочерними)

## Определение

**Hub Page** — категория, которая объединяет дочерние подкатегории. Работает как "диспетчер": продаёт системный подход и разводит трафик по подкатегориям.

## Два типа хабов

| Тип | Определение | Интент | Пример |
|-----|-------------|--------|--------|
| **L1 Hub** | `parent_id = null` | Навигационный | aksessuary, moyka-i-eksterer |
| **L2 Mini-Hub** | `parent_id ≠ null` + есть дочерние L3 | Коммерческий + навигационный | sredstva-dlya-kozhi, avtoshampuni |

## Как определить тип страницы

```
1. Читаем _clean.json → parent_id
   │
   ├─ null → L1 HUB
   │
   └─ not null → проверяем дочерние
                 │
                 ├─ есть дочерние → L2 MINI-HUB
                 │
                 └─ нет дочерних → BUYER GUIDE
```

**Где смотреть дочерние:**
- `tasks/reports/STRUCTURE_TREE.md` — визуальное дерево
- `data/catalog_structure.json` — JSON со всеми связями

## Примеры из структуры

```
L1: Уход за интерьером (HUB — parent_id: null)
  ├── L2: Средства для кожи (MINI-HUB — есть дочерние)
  │   ├── L3: Уход за кожей (BUYER GUIDE)
  │   └── L3: Очистители кожи (BUYER GUIDE)
  ├── L2: Нейтрализаторы запаха (BUYER GUIDE — нет дочерних)
  └── L2: Полироли для пластика (BUYER GUIDE — нет дочерних)
```

## ⛔️ СТОП-ФАКТОРЫ для Hub Pages

### Запрет на каннибализацию

НЕ писать подробные гайды по подкатегориям:
- ❌ "Как выбрать микрофибру" → это контент для L2 mikrofibra-i-tryapki
- ❌ "Виды кистей" → это контент для L2 shchetki-i-kisti
- ❌ "Как стирать микрофибру" → это FAQ для L2, не для Hub

### Запрет на H2 с прямыми ключами подкатегорий

| ❌ Плохо | ✅ Хорошо |
|---------|----------|
| H2: Микрофибра | H2: Текстиль для сушки и полировки |
| H2: Ведра | H2: Организация безопасной мойки |
| H2: Щётки и кисти | H2: Детейлинг колёс и сложных зон |
| H2: Распылители | H2: Работа с химией и защитными составами |

### Запрет на отсутствующие товары

Перед написанием проверить `data/catalog_structure.json` — какие подкатегории реально есть. НЕ упоминать товары из других веток каталога.

## Workflow для Hub Page

```
1. Определить тип    → Проверить parent_id в _clean.json
2. Если parent_id = null → использовать этот шаблон (Hub Page)
3. Загрузить дерево  → tasks/reports/STRUCTURE_TREE.md или catalog_structure.json
4. Найти подкатегории → Все записи с parent_id = {slug}
5. Писать по процессам → H2 = этапы работ, НЕ названия подкатегорий
6. Проверить FAQ     → Вопросы из micro_intents, без каннибализации L2
```

# L2 Mini-Hub (категория с дочерними L3)

## Определение

**L2 Mini-Hub** — категория с `parent_id ≠ null`, у которой есть дочерние L3.

**Ключевое отличие от L1:** интент **коммерческий + навигационный** (не чисто навигационный).

## Как определить L2 Mini-Hub

```
1. _clean.json → parent_id ≠ null
2. tasks/reports/STRUCTURE_TREE.md → есть вложенные L3
```

**Примеры L2 Mini-Hub:**
- sredstva-dlya-kozhi → L3: ochistiteli-kozhi, ukhod-za-kozhey
- avtoshampuni → L3: aktivnaya-pena, shampuni-dlya-ruchnoy-moyki
- sredstva-dlya-stekol → L3: omyvatel, antidozhd, ochistiteli-stekol

**НЕ Mini-Hub (нет дочерних):**
- neytralizatory-zapakha (L2 без L3)
- poliroli-dlya-plastika (L2 без L3)

---

## ⛔️ СТОП-ФАКТОРЫ для L2 Mini-Hub

### Запрет на каннибализацию L3

НЕ писать детальные buyer guide по темам дочерних L3:

| ❌ Каннибализация | ✅ Правильно |
|-------------------|--------------|
| Детальный выбор очистителей (pH, формы) | Общий обзор: очистка — это этап 1 |
| Сравнение кондиционеров и кремов | Общий обзор: защита — это этап 2 |
| FAQ "какой очиститель лучше" | FAQ "что сначала — очистка или защита" |

### Запрет на дублирование FAQ

Если FAQ есть в L3 — НЕ дублировать в L2.

| Уровень | FAQ |
|---------|-----|
| L2 (Mini-Hub) | Общие вопросы про систему/процесс |
| L3 (Buyer Guide) | Детальные вопросы про выбор конкретного товара |
