# Hub Pages (L1 категории)

## Определение

**Hub Page** — родительская категория без `parent_id` (L1 уровень). Работает как "диспетчер": не продаёт конкретный товар, а продаёт системный подход и разводит трафик по подкатегориям.

## Как определить Hub Page

```python
# В _clean.json:
{
  "parent_id": null  # ← Hub Page (L1)
}

# Или проверить в catalog_structure.json:
# Категория без parent_id = Hub
```

**Примеры L1 (Hub):**
- aksessuary
- moyka-i-eksterer
- polirovka
- ukhod-za-intererom
- zashchitnye-pokrytiya

**Примеры L2/L3 (НЕ Hub):**
- mikrofibra-i-tryapki (parent: aksessuary)
- cherniteli-shin (parent: sredstva-dlya-diskov-i-shin)

## ⛔️ СТОП-ФАКТОРЫ для Hub Pages

### Запрет на каннибализацию

НЕ писать подробные гайды по подкатегориям:
- ❌ "Как выбрать микрофибру" → это контент для L2 mikrofibra-i-tryapki
- ❌ "Виды кистей" → это контент для L2 shchetki-i-kisti
- ❌ "Как стирать микрофибру" → это FAQ для L2, не для Hub

### Запрет на H2 с прямыми ключами подкатегорий

| ❌ Плохо | ✅ Хорошо |
|---------|----------|
| H2: Микрофибра | H2: Текстиль для сушки и полировки |
| H2: Ведра | H2: Организация безопасной мойки |
| H2: Щётки и кисти | H2: Детейлинг колёс и сложных зон |
| H2: Распылители | H2: Работа с химией и защитными составами |

### Запрет на отсутствующие товары

Перед написанием проверить `data/catalog_structure.json` — какие подкатегории реально есть. НЕ упоминать товары из других веток каталога.

## Структура Hub Page

```markdown
# {H1 — название категории, БЕЗ "Купить"}

{Intro 30-50 слов: экосистема/системный подход + главная боль + разница между бытовым и профессиональным}

## {H2 по процессу/этапу работ, НЕ по типу товара}

{Описание роли группы товаров в процессе. Без деталей выбора конкретного товара.}

## {H2 по следующему процессу}

{...}

## {Таблица сравнения: Бытовой подход vs Профессиональный}

| Бытовой | Профессиональный |
|---------|------------------|
| ... | ... |

## FAQ

### {Вопрос про выбор/экосистему — из RESEARCH_DATA.md или шаблон}
{Ответ 2-3 предложения}
```

## Параметры Hub Page

| Параметр | Значение |
|----------|----------|
| Объём | 2000-2500 знаков б/п |
| Intro | 30-50 слов |
| H2 | По процессам, НЕ по товарам + secondary keyword |
| Таблица | Сравнение подходов (ошибки vs правильно) |
| FAQ | Из RESEARCH_DATA.md или шаблоны, общие вопросы про экосистему |
| Плотность | ≤2.0% |

---

## Workflow для Hub Page

```
1. Определить тип    → Проверить parent_id в _clean.json
2. Если parent_id = null → использовать этот шаблон (Hub Page)
3. Загрузить дерево  → tasks/reports/STRUCTURE_TREE.md или catalog_structure.json
4. Найти подкатегории → Все записи с parent_id = {slug}
5. Писать по процессам → H2 = этапы работ + secondary keyword, НЕ названия подкатегорий
6. Проверить FAQ     → Вопросы про экосистему из RESEARCH_DATA.md, без каннибализации L2
```

## Checklist для Hub Page

- [ ] `parent_id = null` в _clean.json
- [ ] H2 по процессам + secondary keyword, НЕ по типам товаров
- [ ] Нет детальных гайдов по выбору конкретных товаров
- [ ] FAQ из RESEARCH_DATA.md или шаблоны, общие вопросы про экосистему
- [ ] Нет FAQ, которые должны быть на L2 (как стирать микрофибру → L2)
- [ ] Упомянуты только товары из подкатегорий этой ветки
- [ ] 2000-2500 знаков б/п
