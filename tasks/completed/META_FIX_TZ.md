# ТЗ: Исправление мета-файлов RU категорий

**Дата:** 2026-01-07
**Источник:** validate_meta.py v17.0

## Правила (generate-meta skill v12.0)

### Title (30-60 chars до | Ultimate)

-   **keywords[0] ДОСЛОВНО** в начале — НЕ МЕНЯТЬ!
-   Формула: `{keywords[0]} — купить, цены | Ultimate`
-   Если Title < 30 chars — добавить уточнение ПОСЛЕ keywords[0]

### Description (100-160 chars)

-   Формула: `{keywords[0]} от производителя Ultimate. {Типы}. {Формы/Объёмы}. {Назначение}. Опт и розница.`
-   Типы/формы/объёмы брать из \_clean.json (entities) или PRODUCTS_LIST.md
-   ОБЯЗАТЕЛЬНО: "от производителя Ultimate" + "Опт и розница"

### H1

-   = keywords[0] ДОСЛОВНО
-   БЕЗ "Купить"

## Шаблон Description

```
{keywords[0]} от производителя Ultimate. {Типы по составу/назначению}. {Формы: готовые/концентраты}. {Объёмы: 0.5, 1, 5л}. Опт и розница.
```

**Пример (antibitum - PASS):**

```
Антибитум от производителя Ultimate. Сольвентная основа. Удаление битума, смолы, гудрона. Готовые составы 0.5, 5л. Опт и розница.
```

(129 chars)

## Источники данных для заполнения

1. `categories/{slug}/data/{slug}_clean.json` — entities, keywords
2. `data/generated/PRODUCTS_LIST.md` — реальные товары, типы, объёмы
3. `data/generated/STRUCTURE.md` — структура категорий

## Чеклист перед исправлением

-   [ ] Прочитать \_clean.json для категории
-   [ ] Найти товары в PRODUCTS_LIST.md
-   [ ] Определить: типы, формы, объёмы
-   [ ] Сформировать Description по шаблону
-   [ ] Проверить длину (100-160 chars)
-   [ ] НЕ менять Title и H1 (только если < 30 chars)
-   [ ] Запустить validate_meta.py
